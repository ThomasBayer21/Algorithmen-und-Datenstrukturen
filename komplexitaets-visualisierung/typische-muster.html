Typische Muster für Komplexitätsklassen

Überblick: Diese Seite zeigt typische Code-Muster für verschiedene Komplexitätsklassen. Lerne, welche Strukturen welche Komplexität haben, sowohl für Zeit- als auch für Platzkomplexität.
O(1) - Konstante Zeit
Ausgezeichnet
# Array-Zugriff def get_first(arr): return arr[0] # O(1) # Hash-Table Zugriff def get_value(map, key): return map.get(key) # O(1) # Mathematische Berechnung def is_even(n): return n % 2 == 0 # O(1)
Erkennung: Keine Schleifen, keine Rekursion, direkter Zugriff auf Daten. Die Operation dauert immer gleich lang, unabhängig von der Eingabegröße.

Typische Anwendungen:

  • Array-Indexzugriff
  • Hash-Table Operationen (get, set)
  • Stack push/pop
  • Queue enqueue/dequeue
  • Mathematische Berechnungen
O(log n) - Logarithmische Zeit
Ausgezeichnet
# Binäre Suche def binary_search(arr, target): left, right = 0, len(arr) - 1 while left <= right: mid = (left + right) // 2 if arr[mid] == target: return mid if arr[mid] < target: left = mid + 1 else: right = mid - 1 return -1 # Baum-Höhe berechnen (balanciert) def tree_height(node): if not node: return 0 return 1 + max( tree_height(node.left), tree_height(node.right) )
Erkennung: Suchraum wird in jedem Schritt halbiert oder die Eingabe wird durch einen konstanten Faktor geteilt. Oft bei divide-and-conquer Algorithmen oder balancierten Baumstrukturen.

Typische Anwendungen:

  • Binäre Suche in sortierten Arrays
  • Operationen auf balancierten Bäumen (BST, AVL, Red-Black)
  • Heap Operationen (insert, delete)
  • Suche in Skip Lists
O(n) - Lineare Zeit
Gut
# Array durchlaufen def find_max(arr): max_val = arr[0] for i in range(1, len(arr)): if arr[i] > max_val: max_val = arr[i] return max_val # Lineare Suche def linear_search(arr, target): for i in range(len(arr)): if arr[i] == target: return i return -1 # Summe berechnen def sum_array(arr): total = 0 for num in arr: total += num return total
Erkennung: Eine einzige Schleife über die Eingabe. Jedes Element wird genau einmal besucht. Die Laufzeit wächst direkt proportional zur Eingabegröße.

Typische Anwendungen:

  • Array durchlaufen
  • Lineare Suche
  • Maximum/Minimum finden
  • Liste umkehren
  • String-Operationen (einzelner Durchlauf)
O(n log n) - Linearithmische Zeit
Akzeptabel
# Merge Sort def merge_sort(arr): if len(arr) <= 1: return arr mid = len(arr) // 2 left = merge_sort(arr[:mid]) right = merge_sort(arr[mid:]) return merge(left, right) # Quick Sort (average case) def quick_sort(arr): if len(arr) <= 1: return arr pivot = arr[len(arr) // 2] left = [x for x in arr if x < pivot] middle = [x for x in arr if x == pivot] right = [x for x in arr if x > pivot] return quick_sort(left) + middle + quick_sort(right)
Erkennung: Divide-and-Conquer mit linearer Arbeit in jedem Level. Die Eingabe wird log(n) mal geteilt, und auf jedem Level wird O(n) Arbeit geleistet.

Typische Anwendungen:

  • Effiziente Sortieralgorithmen (Merge Sort, Quick Sort, Heap Sort)
  • Viele Divide-and-Conquer Algorithmen
  • FFT (Fast Fourier Transform)
O(n²) - Quadratische Zeit
Schlecht
# Bubble Sort def bubble_sort(arr): for i in range(len(arr)): for j in range(len(arr) - i - 1): if arr[j] > arr[j + 1]: arr[j], arr[j + 1] = arr[j + 1], arr[j] return arr # Alle Paare finden def find_pairs(arr): pairs = [] for i in range(len(arr)): for j in range(i + 1, len(arr)): pairs.append([arr[i], arr[j]]) return pairs
Erkennung: Zwei verschachtelte Schleifen über die Eingabe. Jedes Element wird mit jedem anderen Element verglichen. Wird schnell unpraktikabel bei großen Datenmengen.

Typische Anwendungen:

  • Einfache Sortieralgorithmen (Bubble, Insertion, Selection Sort)
  • Naive Duplikat-Suche
  • Vergleich aller Paare
  • Matrix-Transposition
O(2ⁿ) - Exponentiell
Sehr schlecht
# Naive Fibonacci def fibonacci(n): if n <= 1: return n return fibonacci(n - 1) + fibonacci(n - 2) # Alle Teilmengen generieren def get_all_subsets(arr): if len(arr) == 0: return [[]] first = arr[0] rest = arr[1:] subsets_without_first = get_all_subsets(rest) subsets_with_first = [ [first] + subset for subset in subsets_without_first ] return subsets_without_first + subsets_with_first
Erkennung: Rekursive Aufrufe verzweigen mehrfach ohne Memoization. Die Anzahl der Operationen verdoppelt sich mit jedem zusätzlichen Input-Element. Wird schnell unpraktikabel.

Typische Anwendungen:

  • Naive rekursive Fibonacci
  • Generierung aller Teilmengen
  • Türme von Hanoi
  • Brute-Force Suche (ohne Pruning)
O(1) - Konstanter Speicher
Ausgezeichnet
# In-Place Vertauschung def swap(arr, i, j): temp = arr[i] # O(1) Speicher arr[i] = arr[j] arr[j] = temp # Iteration mit fixen Variablen def sum_array(arr): total = 0 # O(1) Speicher for num in arr: total += num return total
Erkennung: Nur eine feste Anzahl von Variablen wird verwendet, unabhängig von der Eingabegröße. Keine dynamischen Datenstrukturen.

Typische Anwendungen:

  • In-Place Algorithmen
  • Iterative Berechnungen mit fixen Variablen
  • Bubble Sort, Insertion Sort (in-place)
O(log n) - Logarithmischer Speicher
Ausgezeichnet
# Rekursive binäre Suche (Call Stack) def binary_search_recursive(arr, target, left=0, right=None): if right is None: right = len(arr) - 1 if left > right: return -1 mid = (left + right) // 2 if arr[mid] == target: return mid if arr[mid] < target: return binary_search_recursive(arr, target, mid + 1, right) return binary_search_recursive(arr, target, left, mid - 1) # Rekursionstiefe: log(n) → O(log n) Speicher
Erkennung: Rekursive Algorithmen, bei denen die Rekursionstiefe logarithmisch zur Eingabegröße ist. Der Call Stack wächst nur logarithmisch.

Typische Anwendungen:

  • Rekursive binäre Suche
  • Balancierte Baum-Traversierung
  • Divide-and-Conquer mit einzelnem Zweig
O(n) - Linearer Speicher
Gut
# Neues Array erstellen def double_array(arr): result = [] # O(n) Speicher for num in arr: result.append(num * 2) return result # Hash-Set für Duplikaterkennung def has_duplicates(arr): seen = set() # O(n) Speicher for num in arr: if num in seen: return True seen.add(num) return False # Rekursive Funktion mit linearer Tiefe def sum_recursive(arr, i=0): if i == len(arr): return 0 return arr[i] + sum_recursive(arr, i + 1) # Rekursionstiefe: n → O(n) Speicher
Erkennung: Eine zusätzliche Datenstruktur wird angelegt, die proportional zur Eingabegröße wächst. Oder Rekursionstiefe ist linear.

Typische Anwendungen:

  • Kopie eines Arrays/Listen erstellen
  • Hash-Table/Set zur Duplikaterkennung
  • Merge Sort (zusätzliches Array)
  • BFS/DFS mit Queue/Stack
O(n²) - Quadratischer Speicher
Schlecht
# 2D Matrix erstellen def create_matrix(n): matrix = [] for i in range(n): row = [] for j in range(n): row.append(i * j) matrix.append(row) return matrix # O(n²) Speicher # Alle Paare speichern def store_pairs(arr): pairs = [] for i in range(len(arr)): for j in range(i + 1, len(arr)): pairs.append([arr[i], arr[j]]) # O(n²) Elemente return pairs
Erkennung: Mehrdimensionale Datenstrukturen oder Speicherung aller Kombinationen. Die Speichernutzung wächst quadratisch.

Typische Anwendungen:

  • 2D Arrays/Matrizen
  • Adjacency Matrix für Graphen
  • Dynamic Programming Tables
  • Speicherung aller Paare
O(2ⁿ) - Exponentieller Speicher
Sehr schlecht
# Alle Teilmengen speichern def get_all_subsets(arr): if len(arr) == 0: return [[]] first = arr[0] rest = arr[1:] subsets_without_first = get_all_subsets(rest) subsets_with_first = [ [first] + subset for subset in subsets_without_first ] return subsets_without_first + subsets_with_first # 2ⁿ Teilmengen → O(2ⁿ) Speicher
Erkennung: Speicherung aller möglichen Kombinationen oder Teilmengen. Die Anzahl der gespeicherten Elemente wächst exponentiell.

Typische Anwendungen:

  • Generierung aller Teilmengen
  • Generierung aller Permutationen
  • Brute-Force Suche mit Speicherung aller Pfade

Praktische Richtlinien

Für Zeitkomplexität:

  • O(1), O(log n), O(n): Excellent für große Datenmengen
  • O(n log n): Standard für effizientes Sortieren
  • O(n²): Akzeptabel für kleine Datenmengen (n < 1000)
  • O(n³): Nur für sehr kleine Datenmengen (n < 100)
  • O(2ⁿ): Nur für winzige Datenmengen (n < 20)

Für Platzkomplexität:

  • O(1): Ideal, keine zusätzliche Speichernutzung
  • O(log n): Sehr gut, minimaler zusätzlicher Speicher
  • O(n): Standard, akzeptabel für die meisten Anwendungen
  • O(n²) und höher: Problematisch bei großen Datenmengen
