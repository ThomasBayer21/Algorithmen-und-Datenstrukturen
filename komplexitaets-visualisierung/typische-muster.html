Typische Muster für Komplexitätsklassen

Überblick: Diese Seite zeigt typische Code-Muster für verschiedene Komplexitätsklassen. Lerne, welche Strukturen welche Komplexität haben, sowohl für Zeit- als auch für Platzkomplexität.
O(1) - Konstante Zeit
Ausgezeichnet
// Array-Zugriff function getFirst(arr) { return arr[0]; // O(1) } // Hash-Table Zugriff function getValue(map, key) { return map.get(key); // O(1) } // Mathematische Berechnung function isEven(n) { return n % 2 === 0; // O(1) }
Erkennung: Keine Schleifen, keine Rekursion, direkter Zugriff auf Daten. Die Operation dauert immer gleich lang, unabhängig von der Eingabegröße.

Typische Anwendungen:

  • Array-Indexzugriff
  • Hash-Table Operationen (get, set)
  • Stack push/pop
  • Queue enqueue/dequeue
  • Mathematische Berechnungen
O(log n) - Logarithmische Zeit
Ausgezeichnet
// Binäre Suche function binarySearch(arr, target) { let left = 0, right = arr.length - 1; while (left <= right) { let mid = Math.floor((left + right) / 2); if (arr[mid] === target) return mid; if (arr[mid] < target) left = mid + 1; else right = mid - 1; } return -1; } // Baum-Höhe berechnen (balanciert) function treeHeight(node) { if (!node) return 0; return 1 + Math.max( treeHeight(node.left), treeHeight(node.right) ); }
Erkennung: Suchraum wird in jedem Schritt halbiert oder die Eingabe wird durch einen konstanten Faktor geteilt. Oft bei divide-and-conquer Algorithmen oder balancierten Baumstrukturen.

Typische Anwendungen:

  • Binäre Suche in sortierten Arrays
  • Operationen auf balancierten Bäumen (BST, AVL, Red-Black)
  • Heap Operationen (insert, delete)
  • Suche in Skip Lists
O(n) - Lineare Zeit
Gut
// Array durchlaufen function findMax(arr) { let max = arr[0]; for (let i = 1; i < arr.length; i++) { if (arr[i] > max) max = arr[i]; } return max; } // Lineare Suche function linearSearch(arr, target) { for (let i = 0; i < arr.length; i++) { if (arr[i] === target) return i; } return -1; } // Summe berechnen function sum(arr) { let total = 0; for (const num of arr) { total += num; } return total; }
Erkennung: Eine einzige Schleife über die Eingabe. Jedes Element wird genau einmal besucht. Die Laufzeit wächst direkt proportional zur Eingabegröße.

Typische Anwendungen:

  • Array durchlaufen
  • Lineare Suche
  • Maximum/Minimum finden
  • Liste umkehren
  • String-Operationen (einzelner Durchlauf)
O(n log n) - Linearithmische Zeit
Akzeptabel
// Merge Sort function mergeSort(arr) { if (arr.length <= 1) return arr; const mid = Math.floor(arr.length / 2); const left = mergeSort(arr.slice(0, mid)); const right = mergeSort(arr.slice(mid)); return merge(left, right); } // Quick Sort (average case) function quickSort(arr) { if (arr.length <= 1) return arr; const pivot = arr[Math.floor(arr.length / 2)]; const left = arr.filter(x => x < pivot); const middle = arr.filter(x => x === pivot); const right = arr.filter(x => x > pivot); return [...quickSort(left), ...middle, ...quickSort(right)]; }
Erkennung: Divide-and-Conquer mit linearer Arbeit in jedem Level. Die Eingabe wird log(n) mal geteilt, und auf jedem Level wird O(n) Arbeit geleistet.

Typische Anwendungen:

  • Effiziente Sortieralgorithmen (Merge Sort, Quick Sort, Heap Sort)
  • Viele Divide-and-Conquer Algorithmen
  • FFT (Fast Fourier Transform)
O(n²) - Quadratische Zeit
Schlecht
// Bubble Sort function bubbleSort(arr) { for (let i = 0; i < arr.length; i++) { for (let j = 0; j < arr.length - i - 1; j++) { if (arr[j] > arr[j + 1]) { [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]]; } } } return arr; } // Alle Paare finden function findPairs(arr) { const pairs = []; for (let i = 0; i < arr.length; i++) { for (let j = i + 1; j < arr.length; j++) { pairs.push([arr[i], arr[j]]); } } return pairs; }
Erkennung: Zwei verschachtelte Schleifen über die Eingabe. Jedes Element wird mit jedem anderen Element verglichen. Wird schnell unpraktikabel bei großen Datenmengen.

Typische Anwendungen:

  • Einfache Sortieralgorithmen (Bubble, Insertion, Selection Sort)
  • Naive Duplikat-Suche
  • Vergleich aller Paare
  • Matrix-Transposition
O(2ⁿ) - Exponentiell
Sehr schlecht
// Naive Fibonacci function fibonacci(n) { if (n <= 1) return n; return fibonacci(n - 1) + fibonacci(n - 2); } // Alle Teilmengen generieren function getAllSubsets(arr) { if (arr.length === 0) return [[]]; const first = arr[0]; const rest = arr.slice(1); const subsetsWithoutFirst = getAllSubsets(rest); const subsetsWithFirst = subsetsWithoutFirst.map( subset => [first, ...subset] ); return [...subsetsWithoutFirst, ...subsetsWithFirst]; }
Erkennung: Rekursive Aufrufe verzweigen mehrfach ohne Memoization. Die Anzahl der Operationen verdoppelt sich mit jedem zusätzlichen Input-Element. Wird schnell unpraktikabel.

Typische Anwendungen:

  • Naive rekursive Fibonacci
  • Generierung aller Teilmengen
  • Türme von Hanoi
  • Brute-Force Suche (ohne Pruning)
O(1) - Konstanter Speicher
Ausgezeichnet
// In-Place Vertauschung function swap(arr, i, j) { const temp = arr[i]; // O(1) Speicher arr[i] = arr[j]; arr[j] = temp; } // Iteration mit fixen Variablen function sumArray(arr) { let sum = 0; // O(1) Speicher for (const num of arr) { sum += num; } return sum; }
Erkennung: Nur eine feste Anzahl von Variablen wird verwendet, unabhängig von der Eingabegröße. Keine dynamischen Datenstrukturen.

Typische Anwendungen:

  • In-Place Algorithmen
  • Iterative Berechnungen mit fixen Variablen
  • Bubble Sort, Insertion Sort (in-place)
O(log n) - Logarithmischer Speicher
Ausgezeichnet
// Rekursive binäre Suche (Call Stack) function binarySearchRecursive(arr, target, left = 0, right = arr.length - 1) { if (left > right) return -1; const mid = Math.floor((left + right) / 2); if (arr[mid] === target) return mid; if (arr[mid] < target) return binarySearchRecursive(arr, target, mid + 1, right); return binarySearchRecursive(arr, target, left, mid - 1); } // Rekursionstiefe: log(n) → O(log n) Speicher
Erkennung: Rekursive Algorithmen, bei denen die Rekursionstiefe logarithmisch zur Eingabegröße ist. Der Call Stack wächst nur logarithmisch.

Typische Anwendungen:

  • Rekursive binäre Suche
  • Balancierte Baum-Traversierung
  • Divide-and-Conquer mit einzelnem Zweig
O(n) - Linearer Speicher
Gut
// Neues Array erstellen function doubleArray(arr) { const result = []; // O(n) Speicher for (const num of arr) { result.push(num * 2); } return result; } // Hash-Set für Duplikaterkennung function hasDuplicates(arr) { const seen = new Set(); // O(n) Speicher for (const num of arr) { if (seen.has(num)) return true; seen.add(num); } return false; } // Rekursive Funktion mit linearer Tiefe function sumRecursive(arr, i = 0) { if (i === arr.length) return 0; return arr[i] + sumRecursive(arr, i + 1); } // Rekursionstiefe: n → O(n) Speicher
Erkennung: Eine zusätzliche Datenstruktur wird angelegt, die proportional zur Eingabegröße wächst. Oder Rekursionstiefe ist linear.

Typische Anwendungen:

  • Kopie eines Arrays/Listen erstellen
  • Hash-Table/Set zur Duplikaterkennung
  • Merge Sort (zusätzliches Array)
  • BFS/DFS mit Queue/Stack
O(n²) - Quadratischer Speicher
Schlecht
// 2D Matrix erstellen function createMatrix(n) { const matrix = []; for (let i = 0; i < n; i++) { matrix[i] = []; for (let j = 0; j < n; j++) { matrix[i][j] = i * j; } } return matrix; // O(n²) Speicher } // Alle Paare speichern function storePairs(arr) { const pairs = []; for (let i = 0; i < arr.length; i++) { for (let j = i + 1; j < arr.length; j++) { pairs.push([arr[i], arr[j]]); // O(n²) Elemente } } return pairs; }
Erkennung: Mehrdimensionale Datenstrukturen oder Speicherung aller Kombinationen. Die Speichernutzung wächst quadratisch.

Typische Anwendungen:

  • 2D Arrays/Matrizen
  • Adjacency Matrix für Graphen
  • Dynamic Programming Tables
  • Speicherung aller Paare
O(2ⁿ) - Exponentieller Speicher
Sehr schlecht
// Alle Teilmengen speichern function getAllSubsets(arr) { if (arr.length === 0) return [[]]; const first = arr[0]; const rest = arr.slice(1); const subsetsWithoutFirst = getAllSubsets(rest); const subsetsWithFirst = subsetsWithoutFirst.map( subset => [first, ...subset] ); return [...subsetsWithoutFirst, ...subsetsWithFirst]; } // 2ⁿ Teilmengen → O(2ⁿ) Speicher
Erkennung: Speicherung aller möglichen Kombinationen oder Teilmengen. Die Anzahl der gespeicherten Elemente wächst exponentiell.

Typische Anwendungen:

  • Generierung aller Teilmengen
  • Generierung aller Permutationen
  • Brute-Force Suche mit Speicherung aller Pfade

Praktische Richtlinien

Für Zeitkomplexität:

  • O(1), O(log n), O(n): Excellent für große Datenmengen
  • O(n log n): Standard für effizientes Sortieren
  • O(n²): Akzeptabel für kleine Datenmengen (n < 1000)
  • O(n³): Nur für sehr kleine Datenmengen (n < 100)
  • O(2ⁿ): Nur für winzige Datenmengen (n < 20)

Für Platzkomplexität:

  • O(1): Ideal, keine zusätzliche Speichernutzung
  • O(log n): Sehr gut, minimaler zusätzlicher Speicher
  • O(n): Standard, akzeptabel für die meisten Anwendungen
  • O(n²) und höher: Problematisch bei großen Datenmengen
