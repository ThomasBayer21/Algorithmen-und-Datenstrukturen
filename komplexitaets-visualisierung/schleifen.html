Schleifen & Auswirkungen auf die Komplexität

Überblick: Schleifen sind einer der wichtigsten Faktoren für die Zeitkomplexität von Algorithmen. Diese Seite zeigt verschiedene Schleifenmuster und wie sie die Komplexität beeinflussen.

Häufige Schleifenmuster

1. Einfache Schleife O(n)
for (let i = 0; i < n; i++) { // Konstante Operation print(i); // O(1) }
Erklärung: Eine einfache Schleife, die n-mal durchläuft und in jedem Durchlauf eine konstante Operation ausführt, hat die Komplexität O(n). Die Schleife skaliert linear mit der Eingabegröße.

2. Verschachtelte Schleifen (2 Ebenen) O(n²)
for (let i = 0; i < n; i++) { // n Iterationen for (let j = 0; j < n; j++) { // n Iterationen pro i print(i, j); // O(1) } }
Erklärung: Zwei verschachtelte Schleifen führen zu O(n²). Die innere Schleife läuft n-mal für jeden der n Durchläufe der äußeren Schleife, also insgesamt n × n = n² Operationen.

3. Verschachtelte Schleifen (3 Ebenen) O(n³)
for (let i = 0; i < n; i++) { // n Iterationen for (let j = 0; j < n; j++) { // n Iterationen pro i for (let k = 0; k < n; k++) { // n Iterationen pro (i,j) print(i, j, k); // O(1) } } }
Erklärung: Drei verschachtelte Schleifen führen zu O(n³). Die innerste Schleife läuft n³-mal insgesamt, was sehr schnell unpraktikabel wird.

4. Abhängige Schleife (Halbierung) O(log n)
for (let i = n; i > 0; i = i / 2) { print(i); // O(1) } // oder äquivalent: for (let i = 1; i < n; i = i * 2) { print(i); }
Erklärung: Wenn der Schleifenzähler in jedem Durchlauf halbiert (oder verdoppelt) wird, haben wir logarithmisches Wachstum O(log n). Die Anzahl der Iterationen ist log₂(n).

5. Abhängige verschachtelte Schleife O(n²) - Triangle
for (let i = 0; i < n; i++) { // n Iterationen for (let j = i; j < n; j++) { // (n-i) Iterationen print(i, j); // O(1) } }
Erklärung: Obwohl die innere Schleife von i abhängt, ist die Gesamtkomplexität immer noch O(n²). Die Anzahl der Operationen ist n + (n-1) + (n-2) + ... + 1 = n(n+1)/2 ≈ n²/2 = O(n²).
