Dynamische Programmierung

Fibonacci-Folge als klassisches Beispiel

Was ist Dynamische Programmierung?

Dynamische Programmierung ist eine Optimierungstechnik, die komplexe Probleme löst, indem sie diese in überlappende Teilprobleme zerlegt und die Ergebnisse speichert, um wiederholte Berechnungen zu vermeiden. Das Schlüsselprinzip: Berechne jedes Teilproblem nur einmal und merke dir das Ergebnis (Memoization).

Die Fibonacci-Folge ist das klassische Beispiel, um den dramatischen Unterschied zwischen naiver Rekursion und dynamischer Programmierung zu demonstrieren.

F(n) = F(n-1) + F(n-2) mit F(0) = 0 und F(1) = 1

1. Naive Rekursion

O(2ⁿ) Zeit, O(n) Raum

Die direkte rekursive Implementierung berechnet dieselben Werte mehrfach. Bei F(5) wird F(3) bereits 2x berechnet, bei F(10) wird F(5) 8x berechnet!

function fib(n) { if (n <= 1) return n; return fib(n - 1) + fib(n - 2); }

2. Top-Down mit Memoization

O(n) Zeit, O(n) Raum

Speichert bereits berechnete Ergebnisse in einem Array oder Map. Jeder Wert wird nur einmal berechnet - dramatische Verbesserung!

function fibMemo(n, memo = {}) { if (n in memo) return memo[n]; if (n <= 1) return n; memo[n] = fibMemo(n-1, memo) + fibMemo(n-2, memo); return memo[n]; }

💡 Python-Tipp: In Python kann man einfach den @cache Decorator aus functools verwenden, der automatisch Memoization implementiert!

3. Bottom-Up Iterativ

O(n) Zeit, O(n) Raum

Baut die Lösung von unten nach oben auf. Vermeidet Rekursion vollständig und berechnet die Werte in der richtigen Reihenfolge.

function fibBottomUp(n) { if (n <= 1) return n; let dp = [0, 1]; for (let i = 2; i <= n; i++) { dp[i] = dp[i-1] + dp[i-2]; } return dp[n]; }

Vergleich der Methoden

Methode Zeitkomplexität Raumkomplexität Vorteile Nachteile
Naive Rekursion O(2ⁿ) O(n) Einfach zu verstehen und implementieren Extrem langsam, viele redundante Berechnungen
Top-Down Memoization O(n) O(n) Intuitive rekursive Struktur, nur nötige Werte berechnet Rekursions-Overhead, Stack-Space
Bottom-Up Iterativ O(n) O(n) oder O(1)* Kein Rekursions-Overhead, kann auf O(1) Raum optimiert werden Muss alle Werte berechnen, auch wenn nicht alle benötigt

* Bei Bottom-Up kann man nur die letzten zwei Werte speichern statt des gesamten Arrays, wodurch die Raumkomplexität auf O(1) reduziert wird.
